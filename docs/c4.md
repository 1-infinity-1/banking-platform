# C4 Диаграммы

C4 Model — иерархия из четырёх уровней абстракции: Context → Container → Component → Code. Диаграммы ниже охватывают первые три уровня.

---

## Уровень 1 — System Context

Кто взаимодействует с системой и какую роль она выполняет в более широкой среде.

```mermaid
C4Context
  title Системный контекст — Banking Platform

  Person(customer, "Клиент", "Конечный пользователь: вход, переводы, просмотр выписок")
  Person(admin, "Администратор", "Управляет пользователями и ролями через API")

  System(banking, "Banking Platform", "Микросервисная банковская платформа на Go: аутентификация, счета, переводы, бухгалтерский журнал")

  Rel(customer, banking, "REST/HTTPS", "JSON over HTTP")
  Rel(admin, banking, "REST/HTTPS", "JSON over HTTP")

  UpdateLayoutConfig($c4ShapeInRow="3", $c4BoundaryInRow="1")
```

## Уровень 2 — Containers

Какие исполняемые единицы входят в систему, как они общаются и где хранят данные.

```mermaid
C4Container
  title Контейнерная диаграмма — Banking Platform

  Person(customer, "Клиент", "Web / Mobile")
  Person(admin, "Администратор", "")

  System_Boundary(banking, "Banking Platform") {

    Container(gateway, "Gateway Service", "Go · ogen/OpenAPI", "REST API Gateway. Единственная точка входа для внешних клиентов. :8081")
    Container(auth, "Auth Service", "Go · gRPC", "Аутентификация, сессии, JWT-токены, роли и permissions. :8082")
    Container(account, "Account Service", "Go · gRPC", "Управление банковскими счетами и балансами. :8083 [скелет]")
    Container(transaction, "Transaction Service", "Go · gRPC + Kafka", "Переводы и пополнения счетов. :8084 [скелет]")
    Container(ledgerGrpc, "Ledger Service (gRPC)", "Go · gRPC", "Выписки по счетам. :8085")
    Container(ledgerConsumer, "Ledger Service (Consumer)", "Go · Kafka consumer", "Потребляет события транзакций, создаёт проводки")

    ContainerDb(authDb, "Auth DB", "PostgreSQL 17", "users · sessions · devices · refresh_tokens · roles · permissions")
    ContainerDb(accountDb, "Account DB", "PostgreSQL 17", "accounts")
    ContainerDb(transactionDb, "Transaction DB", "PostgreSQL 17", "transactions")
    ContainerDb(ledgerDb, "Ledger DB", "PostgreSQL 17", "ledger_entries")
    Container(kafka, "Kafka", "Confluent Kafka 7.8 (KRaft)", "topic: transactions.completed")
  }

  Rel(customer, gateway, "REST/HTTP", ":8081")
  Rel(admin, gateway, "REST/HTTP", ":8081")

  Rel(gateway, auth, "gRPC", ":8082")
  Rel(gateway, account, "gRPC", ":8083")
  Rel(gateway, transaction, "gRPC", ":8084")
  Rel(gateway, ledgerGrpc, "gRPC", ":8085")

  Rel(transaction, account, "gRPC", ":8083")
  Rel(transaction, kafka, "produce", "transactions.completed")
  Rel(kafka, ledgerConsumer, "consume", "transactions.completed")

  Rel(auth, authDb, "pgx/v5", "SQL")
  Rel(account, accountDb, "pgx/v5", "SQL")
  Rel(transaction, transactionDb, "pgx/v5", "SQL")
  Rel(ledgerGrpc, ledgerDb, "pgx/v5", "SQL")
  Rel(ledgerConsumer, ledgerDb, "pgx/v5", "SQL")

  UpdateLayoutConfig($c4ShapeInRow="4", $c4BoundaryInRow="1")
```

## Уровень 3 — Components: Gateway Service

Из каких компонентов состоит `gateway-service` — самый сложный сервис с fan-out к остальным.

```mermaid
C4Component
  title Компонентная диаграмма — Gateway Service

  Person_Ext(client, "Клиент", "HTTP requests")

  Container_Boundary(gw, "Gateway Service") {
    Component(traceMW, "Trace Middleware", "Go", "Генерирует / извлекает TraceID и RequestID из заголовков. Кладёт в context.Context")
    Component(logMW, "Logging Middleware", "Go", "Request-level logging: метод, путь, статус, latency, trace_id")
    Component(handlers, "HTTP Handlers", "Go · ogen", "Сгенерированные обработчики из openapi.yaml. Валидируют запросы, маппят → domain types")
    Component(services, "Services", "Go", "Бизнес-оркестрация: вызывает нужный gRPC-клиент, маппит ответы")
    Component(errHandler, "Error Handler", "Go", "GatewayHandler.NewError — маппинг domain errors → HTTP статусы")

    Component(authClient, "Auth gRPC Client", "Go", "Инжектирует trace headers. Маппит proto ↔ domain")
    Component(accountClient, "Account gRPC Client", "Go", "Инжектирует trace headers. Маппит proto ↔ domain")
    Component(txClient, "Transaction gRPC Client", "Go", "Инжектирует trace headers. Маппит proto ↔ domain")
    Component(ledgerClient, "Ledger gRPC Client", "Go", "Инжектирует trace headers. Маппит proto ↔ domain")
  }

  Container_Ext(auth, "Auth Service", "gRPC :8082", "")
  Container_Ext(account, "Account Service", "gRPC :8083", "")
  Container_Ext(tx, "Transaction Service", "gRPC :8084", "")
  Container_Ext(ledger, "Ledger Service", "gRPC :8085", "")

  Rel(client, traceMW, "HTTP/JSON")
  Rel(traceMW, logMW, "")
  Rel(logMW, handlers, "")
  Rel(handlers, services, "domain types")
  Rel(handlers, errHandler, "on error")
  Rel(services, authClient, "")
  Rel(services, accountClient, "")
  Rel(services, txClient, "")
  Rel(services, ledgerClient, "")
  Rel(authClient, auth, "gRPC + metadata")
  Rel(accountClient, account, "gRPC + metadata")
  Rel(txClient, tx, "gRPC + metadata")
  Rel(ledgerClient, ledger, "gRPC + metadata")

  UpdateLayoutConfig($c4ShapeInRow="4", $c4BoundaryInRow="1")
```

## Уровень 3 — Components: Auth Service

Внутренняя структура `auth-service` — наиболее полностью реализованного сервиса.

```mermaid
C4Component
  title Компонентная диаграмма — Auth Service

  Container_Ext(gateway, "Gateway Service", "gRPC calls", "")

  Container_Boundary(authSvc, "Auth Service") {
    Component(interceptors, "gRPC Interceptors", "Go", "Chain: trace → logging → error mapping → recovery")
    Component(authTransport, "Auth Transport", "Go · gRPC", "Login, Logout, RefreshToken handlers")
    Component(mgmtTransport, "Management Transport", "Go · gRPC", "CreateUser handler")
    Component(authService, "Auth Service", "Go", "Логика аутентификации: bcrypt, session lifecycle, token генерация")
    Component(mgmtService, "Management Service", "Go", "Создание пользователей, назначение ролей")
    Component(tokenMgr, "JWT Manager", "Go · golang-jwt", "Генерация и валидация access/refresh JWT-токенов")
    Component(txManager, "Tx Manager", "Go · pgx", "BeginFunc(ctx, fn): управление транзакциями БД")

    Component(userRepo, "User Repository", "Go · pgx", "CRUD пользователей, маппинг DB → domain errors")
    Component(sessionRepo, "Session Repository", "Go · pgx", "CRUD сессий")
    Component(deviceRepo, "Device Repository", "Go · pgx", "CRUD устройств")
    Component(tokenRepo, "Refresh Token Repository", "Go · pgx", "Хранение и отзыв refresh tokens")
  }

  ContainerDb_Ext(db, "Auth DB", "PostgreSQL 17", "users · sessions · devices · refresh_tokens · roles · permissions")

  Rel(gateway, interceptors, "gRPC")
  Rel(interceptors, authTransport, "")
  Rel(interceptors, mgmtTransport, "")
  Rel(authTransport, authService, "domain types")
  Rel(mgmtTransport, mgmtService, "domain types")
  Rel(authService, tokenMgr, "generate/validate")
  Rel(authService, txManager, "BeginFunc")
  Rel(mgmtService, txManager, "BeginFunc")
  Rel(txManager, userRepo, "pgx.Tx")
  Rel(txManager, sessionRepo, "pgx.Tx")
  Rel(txManager, deviceRepo, "pgx.Tx")
  Rel(txManager, tokenRepo, "pgx.Tx")
  Rel(userRepo, db, "SQL")
  Rel(sessionRepo, db, "SQL")
  Rel(deviceRepo, db, "SQL")
  Rel(tokenRepo, db, "SQL")

  UpdateLayoutConfig($c4ShapeInRow="4", $c4BoundaryInRow="1")
```
