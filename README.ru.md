# Banking Platform

> [English](README.md) · [Русский](README.ru.md)

Banking Platform — backend-проект на Go, реализованный в виде микросервисного монорепозитория.  
Проект моделирует базовую банковскую платформу с единой точкой входа для клиентских приложений, сервисом пользователей и авторизации, сервисом счетов, сервисом транзакций и сервисом бухгалтерского журнала.

Цели проекта:
- спроектировать и реализовать микросервисную архитектуру на Go;
- зафиксировать контракты между сервисами;
- проработать схемы данных и бизнес-потоки;
- показать разделение ответственности между сервисами;
- отработать подходы к REST, gRPC, Kafka, idempotency и audit log.

---

# Сервисы

## `gateway-service`
Единая REST/BFF точка входа для web и mobile клиентов. Маршрутизирует запросы во внутренние сервисы, агрегирует данные и обеспечивает внешнюю API-границу системы.

## `auth-service`
Управляет пользователями, аутентификацией, сессиями, ролями и permissions. Выдает и валидирует токены, предоставляет пользовательские данные другим сервисам.

## `account-service`
Отвечает за банковские счета, их статусы и текущие балансы. Является source of truth для денежных остатков.

## `transaction-service`
Оркестрирует денежные операции: переводы и пополнения. Координирует debit/credit сценарии, хранит статусы транзакций и публикует события о завершенных операциях.

## `ledger-service`
Ведет бухгалтерский и аудиторский журнал операций. Асинхронно получает события транзакций и формирует выписки по счетам.

# Архитектура

Система строится вокруг следующих принципов:
- клиентские приложения работают только с `gateway-service` по REST;
- внутреннее синхронное взаимодействие между сервисами выполняется по gRPC;
- асинхронная интеграция выполняется через Kafka;
- `account-service` хранит текущие балансы;
- `ledger-service` хранит неизменяемую историю проводок;
- `transaction-service` управляет бизнес-потоком денежных операций;
- `auth-service` отвечает за аутентификацию и пользовательский контекст доступа.

# Взаимодействие сервисов

| Отправитель | Получатель | Транспорт | Назначение |
|---|---|---|---|
| Клиентские приложения | Gateway Service | REST | Внешний API |
| Gateway Service | Auth Service | gRPC | Аутентификация и данные пользователя |
| Gateway Service | Account Service | gRPC | Счета и балансы |
| Gateway Service | Transaction Service | gRPC | Переводы и история |
| Gateway Service | Ledger Service | gRPC | Выписки |
| Transaction Service | Account Service | gRPC | Списание / зачисление |
| Transaction Service | Kafka | Kafka | Публикация событий о завершённых транзакциях |
| Kafka | Ledger Service | Kafka | Потребление событий транзакций |
